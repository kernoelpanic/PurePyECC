ecgf2m_add_point(x1, y1, x2, y2, f, a=0)

source code 

Add a point to another (x1,y1)+(x2,y2)=(x3,y3)

let J + K = L where L = (x3, y3), then the equations to add two points with different x coodrinates x1 != x2:

   s = (y2 + y1) / (x2 + x1)
   x3 = s^2 + s + x1 + x2 + a
   y3 = s*(x1 + x3) + x3 + y1 = s*(x2 + x3) + x3 + y2
Parameters:
  • x1 (int or long), y1 (int or long) - point one as two binary polynomials
  • x2 (int or long), y2 (int or long) - Point two as two binary polynomials
  • f (int or long) - irreducable polynome f(x), used for modular reduction
  • a (int or long) - Coefficient 'a' of curve, default is a=0. In a Koblitz curve 'a' is either 0 or 1 and 'b' is always zero, b=0.
Returns:
(x3,y3), sum of two points as new point represented as tuple

ecgf2m_dbl_point(x, y, f, a=0)

source code 

Add point to itself aka. double point. (x,y)*2=(x3,y3)

Let 2J = L where L = (x3, y3). The the equations to double the point J = (x, y) are as follows:

   s = x + (y / x)
   x3 = s^2 + s + a = x^2 + b / x^2
   y3 = x^2 + s*x3 + x3
Parameters:
  • x (int or long), y (int or long) - point to double as two binary polynomials
  • f (int or long) - irreducable polynome f(x), used for modular reduction
  • a (int or long) - Coefficient 'a' of curve, default is a=0. In a Koblitz curve 'a' is either 0 or 1 and 'b' is always zero, b=0.
Returns:
(x3,y3), doubled point as tuple

ecgf2m_sub_point(x1, y1, x2, y2, f, a=0)

source code 

Subtraction of two points (x1,y1)-(x2,y2)=(x3,y3)

Let J - K = J + (-K) where -K = (xk,xk + yk):

   (x1,y1) - (x2,y2) = (x1,y1) + (-(x2,y2))
   (x1,y1) - (x2,y2) = (x1,y1) + (x2,x2 + y2))
Parameters:
  • x1 (int or long), y1 (int or long) - Point one as two binary polynomials
  • x2 (int or long), y2 (int or long) - Point two as two binary polynomials
  • f (int or long) - irreducable polynome f(x), used for modular reduction
  • a (int or long) - Coefficient 'a' of curve, default is a=0. In a Koblitz curve 'a' is either 0 or 1 and 'b' is always zero, b=0.
Returns:
(x3,y3), subtracted point as tuple

gf2m_modmulinv(a, f)

source code 

Element Inversion a(x) => a^-1 mod f(x)

Calculates the modular multiplicative inverse of a(x) mod f(x). Ueses an adapted version of the exdendet euclidiean algorithem. The outline for the adapted algorithem was taken from Katsuki Kobayashi et al. 'An Algorithem for Inversion in $GF(2^m)$ Suitable for Implementation Using a Polynomial Multiply Instruction on GF(2)'. The same algorithem can also be found in Hankerson et al. 'Software Implementation of Elliptic Curve Cryptography over Binary Fields', 24.01.2000

Parameters:
  • a (int or long) - binary polynomial
Returns:
ux, modular multiplicative as binary polynomail
